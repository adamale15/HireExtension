# HireExtension - Product Requirements Document

## 1. Overview

HireExtension is a cross-browser extension (Chrome, Firefox, Edge) that enhances the Jobright.ai job search experience. It scans job listings on Jobright pages, matches them against the user's uploaded resumes using Google Gemini AI, and categorizes each job by fit level. It supports multiple resumes, recommends which resume to use per job, offers AI-powered resume tailoring, and helps users find and connect with hiring managers on LinkedIn.

## 2. Problem Statement

Job seekers waste significant time manually evaluating whether they are a good fit for each listing. They often apply to jobs where they have little chance, or miss roles that are strong matches. Managing multiple versions of a resume and tailoring them per application is tedious. Finding the right hiring manager to network with adds even more friction.

HireExtension eliminates this friction by automating job-resume matching, providing clear go/no-go signals, and streamlining outreach -- all directly within the browser.

## 3. Target Users

- Active job seekers using Jobright.ai as their primary job board
- Professionals maintaining multiple resumes for different role types (e.g., frontend vs. fullstack)
- Users who want data-driven guidance on where to invest application effort

## 4. Core Features

### 4.1 Job Scanning (Content Script)

| Attribute         | Detail |
|-------------------|--------|
| **Trigger**       | User navigates to a Jobright search results or recommended jobs page |
| **Pages**         | `jobright.ai/jobs/*` (search results), `jobright.ai/recommend*` (recommended/saved) |
| **Extracted Data**| Job title, company name, location, work model, salary range, job URL, H1B flag, applicant count, posted time |
| **Pagination**    | `MutationObserver` detects new job cards loaded via infinite scroll |
| **Caching**       | Scraped jobs are cached in `chrome.storage.local` keyed by URL to avoid redundant scraping |

### 4.2 Resume Management

| Attribute           | Detail |
|---------------------|--------|
| **Upload Format**   | PDF only |
| **Storage**         | PDF file in Firebase Storage; parsed metadata in Cloud Firestore |
| **Multiple Resumes**| Users can store and name multiple resumes (e.g., "Frontend Resume", "Data Engineer Resume") |
| **Default Resume**  | One resume can be marked as default for matching |
| **Parsing**         | Gemini API extracts: skills, experience history, education, certifications, summary |
| **Operations**      | Upload, rename, delete, set as default, view parsed profile |

### 4.3 AI Job Matching and Categorization

| Attribute               | Detail |
|-------------------------|--------|
| **Engine**              | Google Gemini API with structured JSON output |
| **Input**               | Parsed resume profile(s) + scraped job data (title, company, description, requirements) |
| **Output per job**      | `match_score` (0-100), `category`, `reasons[]`, `recommended_resume_id`, `missing_skills[]` |
| **Categories**          | **Safe Apply** (score >= 75, green), **Moderate Apply** (score 50-74, orange), **Don't Apply** (score < 50, red) |
| **Multi-resume logic**  | When multiple resumes exist, Gemini evaluates all and recommends the best one per job |
| **Concurrency**         | 3-5 jobs processed in parallel to respect API rate limits |
| **Detail page fetch**   | For richer matching, the extension fetches the full job description from the job detail URL |

### 4.4 Resume Tailoring

| Attribute            | Detail |
|----------------------|--------|
| **Trigger**          | User clicks "Tailor Resume" on a specific job card |
| **Constraint**       | Must NOT fabricate experience or skills; only reword and re-emphasize existing qualifications |
| **Output**           | Structured JSON of section-level changes (summary, bullet points, skills order) |
| **Review UI**        | Side-by-side diff view: original vs. tailored, with accept/reject per change |
| **Export**           | Downloadable tailored PDF |
| **Original safety**  | The original resume in Firebase Storage is never modified |

### 4.5 LinkedIn Hiring Manager Finder (Semi-Automated)

| Attribute            | Detail |
|----------------------|--------|
| **Trigger**          | User clicks "Find Hiring Manager" on a job card |
| **Step 1**           | Extension constructs a LinkedIn People Search URL using company name + relevant keywords |
| **Step 2**           | Opens LinkedIn search results in a new tab |
| **Step 3**           | Content script on LinkedIn highlights likely hiring managers and injects a "Connect" button |
| **Step 4**           | Clicking "Connect" pre-fills a personalized connection message generated by Gemini |
| **Step 5**           | User reviews and manually clicks LinkedIn's native "Send" button |
| **Automation level** | Semi-automated -- the extension never sends requests without user confirmation |

### 4.6 Authentication

| Attribute       | Detail |
|-----------------|--------|
| **Provider**    | Firebase Auth with Google Sign-In |
| **Chrome/Edge** | Uses `chrome.identity.getAuthToken()` for seamless OAuth |
| **Firefox**     | Falls back to web-based OAuth popup flow |
| **Session**     | Persisted in `chrome.storage.local`; auto-refreshed on expiry |
| **Purpose**     | Syncs resumes and preferences across devices |

## 5. Non-Functional Requirements

### 5.1 Performance

- Job scraping must complete within 2 seconds of page load
- AI matching should display first results within 5 seconds (streaming/progressive)
- Side panel must open in under 1 second
- Cached jobs load instantly from local storage

### 5.2 Security

- Gemini API key is provided by the user and stored encrypted in `chrome.storage.local`
- Firebase config is public (security enforced via Firestore security rules)
- No user data is sent to any server other than Firebase and Google Gemini API
- Resume PDFs are stored under authenticated user paths in Firebase Storage

### 5.3 Privacy

- Extension only activates on `jobright.ai` and `linkedin.com` domains
- No browsing data is collected outside these domains
- Users can delete all their data (resumes, history) from the Settings tab

### 5.4 Cross-Browser Compatibility

- Chrome 120+, Firefox 115+, Edge 120+
- WXT framework handles Manifest V2/V3 differences
- Side panel: native `chrome.sidePanel` on Chrome/Edge; sidebar fallback on Firefox
- All `chrome.*` APIs accessed through WXT's `browser` polyfill

## 6. Data Models

### 6.1 User

```
{
  uid: string,
  email: string,
  displayName: string,
  photoURL: string,
  geminiApiKey: string (encrypted),
  defaultResumeId: string,
  createdAt: timestamp,
  updatedAt: timestamp
}
```

### 6.2 Resume

```
{
  id: string,
  userId: string,
  name: string,                    // user-assigned label
  fileName: string,                // original PDF filename
  pdfUrl: string,                  // Firebase Storage download URL
  parsedProfile: {
    skills: [{ name: string, category: "technical" | "soft", yearsOfExperience: number }],
    experience: [{ title: string, company: string, startDate: string, endDate: string, bullets: string[] }],
    education: [{ degree: string, field: string, institution: string, year: number }],
    certifications: string[],
    summary: string
  },
  uploadedAt: timestamp,
  updatedAt: timestamp
}
```

### 6.3 ScrapedJob

```
{
  id: string,
  title: string,
  company: string,
  location: string,
  workModel: "remote" | "onsite" | "hybrid",
  salary: { min: number, max: number, currency: string } | null,
  url: string,
  h1bSponsorship: boolean | null,
  applicantCount: string,
  postedAt: string,
  scrapedAt: timestamp
}
```

### 6.4 JobMatch

```
{
  id: string,
  jobId: string,
  userId: string,
  resumeId: string,                // resume used for this match
  score: number,                   // 0-100
  category: "safe_apply" | "moderate_apply" | "dont_apply",
  reasons: string[],
  missingSkills: string[],
  recommendedResumeId: string,
  matchedAt: timestamp
}
```

### 6.5 TailoredResume

```
{
  id: string,
  originalResumeId: string,
  jobId: string,
  userId: string,
  changes: [{ section: string, original: string, tailored: string, accepted: boolean }],
  tailoredPdfUrl: string | null,
  createdAt: timestamp
}
```

## 7. API and Permissions

### 7.1 Browser Extension Permissions

- `activeTab` -- access current tab content on Jobright/LinkedIn
- `sidePanel` -- open the side panel UI
- `storage` -- local caching of jobs and session
- `identity` -- Google OAuth token for Firebase Auth (Chrome/Edge)
- `host_permissions`: `*://jobright.ai/*`, `*://www.linkedin.com/*`

### 7.2 External APIs

- **Google Gemini API**: resume parsing, job matching, resume tailoring, message generation
- **Firebase Auth**: Google Sign-In
- **Cloud Firestore**: user data, resume metadata, job match history
- **Firebase Storage**: resume PDF file storage

## 8. Out of Scope (v1)

- Support for job boards other than Jobright.ai (LinkedIn Jobs, Indeed, etc.)
- Fully automated LinkedIn connection requests (always semi-automated)
- Cover letter generation
- Application tracking / status management
- Mobile browser support
- Resume creation from scratch (only tailoring of existing resumes)
- DOCX or plain text resume uploads (PDF only in v1)

## 9. Success Metrics

- User can go from installing the extension to seeing categorized jobs in under 3 minutes
- AI matching accuracy validated by user feedback (thumbs up/down on categorization)
- At least 80% of tailored resume suggestions are accepted by users
- Zero instances of fabricated experience in tailored resumes

## 10. Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Jobright changes DOM structure | Scraping breaks | Use resilient selectors; add a version check; notify users when scraper needs update |
| Gemini API rate limits | Matching stalls for large job lists | Batch processing with concurrency cap; cache results; progressive UI updates |
| LinkedIn detects automation | Account restrictions | Semi-automated only; user always clicks "Send"; no automated form submissions |
| Firebase free tier limits | Storage/read quotas exceeded | Aggressive local caching; lazy sync; upgrade path documented |
| Gemini hallucinates skills | Tailored resume contains fabricated content | Explicit prompt constraints; diff review UI; user must accept each change |
